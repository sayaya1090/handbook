# handbook

## Domains

- Document
  여러개의 속성(Attribute)로 정의된 데이터입니다. 참조를 위한 시얼을 가지며, 유효기간 및 버전 관리가 지원됩니다.
- Attribute
    - ValueAttribute:
      단일 값을 가지는 속성을 나타냅니다. 예: 문자열, 숫자 등.
    - ArrayAttribute:
      배열 형태로 여러 값을 저장할 수 있는 속성을 나타냅니다. 예: 사용자 취미 목록.
    - MapAttribute:
      키-값 쌍의 배열로 데이터를 저장하는 속성을 나타냅니다. 예: 환경설정("language"="ko", "timezone"="UTC").
    - DocumentAttribute:
      특정 타입을 가진 다른 문서 참조값입이다. 참조는 시리얼을 사용하여 약한 참조로 되어 있습니다.
    - FileAttribute:
      파일 데이터를 저장하거나 참조할 수 있는 속성을 나타냅니다. 예: 사용자 프로필 사진, 문서 파일.
- Validator
    - Regex:
      정규식 유효성 검사기는 입력 값이 특정 정규식 패턴과 일치하는지 확인합니다. 예: 이메일 형식 검증, 전화번호 형식 확인 등.
    - Number:
      숫자 유효성 검사기는 입력 값이 특정 범위(최소값과 최대값) 내에 있는지, 정수나 소수인지 확인하며, 필요하다면 특정 숫자 패턴과 일치하는지도 검사합니다. 예: 사용자의 나이 또는 제품 가격을
      검증.
    - Date:
      날짜 유효성 검사기는 입력 값이 유효한 날짜 형식인지 확인합니다. 특정 날짜 형식(예: "yyyy-MM-dd")을 따르는지 검사하며, 필요시 과거, 현재, 또는 미래의 날짜인지 여부도 확인할 수 있습니다.
      예: 사용자의 생년월일, 이벤트의 시작일 또는 종료일을 검증.
    - Bool:
      불리언 유효성 검사기는 입력 값이 논리형(true/false)인지 확인합니다. 예: 사용자의 동의 여부 검증.
    - Enum:
      열거형 유효성 검사기는 입력 값이 미리 정의된 값들 중 하나인지 확인합니다. 예: 상태코드("ACTIVE", "INACTIVE") 또는 카테고리 검증.
- Role:
  역할은 사용자나 그룹이 수행할 수 있는 특정 작업이나 책임을 정의합니다. 예: 관리자, 편집자, 게스트 등.
- Group:
  그룹은 사용자들을 모아 하나의 단위로 관리하기 위한 개념입니다. 예: 팀, 부서, 프로젝트 그룹 등.
- User:
  사용자는 시스템에 접근하는 개별 주체를 의미하며, 사용자명과 권한, 역할 등을 통해 정의됩니다.
